§ 02 Findings

Six things that don't survive scrutiny.

Ordered by foundational impact, not by size of the fix. Each cites the exact source location.

01 · SystemHigh

The red Quarantined badge breaks the system in the most visible row.

Line 374 hard-codes background: #fee2e2, color: #dc2626, border-radius: 4px — inline, in the table. Everywhere else we use <Chip> with pill radius. This is the single highest-severity field in the register and it's rendered in a style that exists nowhere else in the app.

FixReplace with <Chip tone="danger"> and lift quarantine out of the name cell into a row-level severity gutter (see redesign).
02 · HierarchyHigh

The detail panel is a drawer of nine section headings with no density.

Identity, Address, Classification, Relations, Performance, Risks, Obligations, Services, Change log — all stacked at the same typographic weight. Five of them render "No data linked yet." — that's 55% of the panel's vertical space spent saying nothing. The user has to scroll past empty sections to find change log.

FixCollapse empty chapters behind a one-line summary; treat identity as primary, everything else as dossier chapters opening only when they have content.
03 · InteractionMedium

The register has no keyboard model, no search, no filter, no sort.

4 rows today, but this is the register — it grows. onClick on <tr> is the only way to select; there's no arrow/enter navigation, no type-to-jump, no column sort. The only filter is a raw <input type="checkbox"> labeled "Show quarantined" with an inline style= prop.

FixInline-filter strip (search + sort + quarantine toggle) sitting in the register header; ↑/↓ navigation; / to focus search; E to edit.
04 · CopywritingMedium

"Not provided", "No data linked yet", "Register link" — three tones in one panel.

optionalText() defaults everything missing to the string "Not provided" (line 120). Meanwhile sibling sections say "No data linked yet." and the action reads "Open official record". We're a record-keeping product; copy should be consistent and earned, not filler.

FixOne missing-value convention: — in tabular density; "Not on file" in dossier chapters; skip sections that have no data entirely.
05 · A11yMedium

The edit toggle is an icon-only button that flips meaning silently.

Lines 445–452 render a <button> whose aria-label flips between "View mode" and "Edit mode" — but the icon swaps between three lines (menu) and a pencil. A screen reader user hears the state change, but sighted users see a visual they don't associate with edit at all. The title attribute is the only tooltip.

FixLabeled pill button: "Edit" (pencil) / "Done editing" (check). Inline edit isn't a mode flip — it's a direct action per field (see redesign).
06 · RethinkOpportunity

This is a register, not a CRUD list. Let it behave like one.

A register is an authoritative document. The current UI treats it as a spreadsheet with a side-drawer for the current row. What if Companies looked and read like a bound dossier — a list on the left, the record as a full page on the right, and every data point anchored by its provenance (as of, changed by, source)?

FixFull-page dossier. List + record, not table + drawer. The redesign below pushes this direction.

§ 05 Interaction notes

How it feels, beyond the screenshot.

The parts that don't fit on a flat mockup.

01Keyboard
/ focuses search. ↑↓ walks the register. ⏎ opens the record; E flips the record into edit. ⌘K for cross-module search stays global.
02Inline edit
Not a mode flip. Hover a row in the dossier, a pencil appears at the right; click writes the value in place. Save/cancel anchor to the edited row, not a footer.
03Empty chapters
Chapters with no data don't render. A faint "+ link performance data" strip at the foot of the dossier lets the user add what's missing; it's a deliberate gap, not a wall of "No data yet."
04Held for review
"Quarantined" reads as sterile. Records that fail validation are held. The tag is one word in a single amber-rose tone; the reason is shown above the dossier as a callout, never inside the field.
05Loading & error
Register list skeletons are three empty rows with the monospace eyebrow "§ syncing". Errors are a single paper-toned strip at the top of the list, never a modal. Optimistic save stays as-written until the server confirms.
06Empty state
Zero companies is not an empty state — it's the welcome screen: a single-line "Open the register" CTA that launches the wizard. We never say "No data".
